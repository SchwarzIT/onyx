Colors enhance the visual appeal and user experience of an interface. They serve as tools to convey information, establish hierarchy and evoke emotions. A consistent color scheme guides users, highlights important elements and creates a cohesive design. Color choices influence how users perceive and interact with the interface.

## Colors

We offer a wide variety of pre-defined color palettes that can be used universally for different use cases and adapt automatically when switched between light and dark mode.

::info-card{headline="Variables vs. hard coded colors"}
We strongly recommend to always the CSS or Figma variables shown below when using colors instead of hard-coding the actual color values. This ensures that the application maintains themeable and adjust correctly between light and dark mode. For more information, refer to our [variables](/introduction/foundation/variables) documentation.
::

## Usage

::content-tabs
#dev
To switch between light and dark mode, define the [color-scheme](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/color-scheme) CSS property on any element that you want to apply the theme to. You can define the property either globally for the whole application or only for specific components.

```css [Apply dark mode]
:root {
  color-scheme: dark;
}
```

To also support older tools and libraries, we additionally support applying the dark mode using a `.dark` CSS class but using the `color-scheme` CSS property is recommended instead.

### Automatic light/dark mode

To support a better user experience, we recommend to support automatically applying the light and dark mode depending on the users preferences / system settings. See below on how to implement this using Vue or Nuxt. Additionally, you can use our [OnyxColorSchemeMenuItem](https://storybook.onyx.schwarz/?path=/docs/navigation-navbar-modules-colorschememenuitem--docs) or [OnyxColorSchemeDialog](https://storybook.onyx.schwarz/?path=/docs/navigation-navbar-modules-colorschemedialog--docs) inside your nav bar to let the user interactively adjust the theme between auto, light and dark mode.

  ::content-tabs
  #vue
  <steps>

    ::step
    #headline
    Install VueUse

    #default
    If not already done, install the [VueUse](https://vueuse.org/) npm package:

    :npm-install-code-tabs{packages="@vueuse/core"}
    ::

    ::step
    #headline
    Enable automatic light/dark mode

    #default
    Add the [useColorMode()](https://vueuse.org/core/useColorMode/) composable from VueUse to easily support the automatic light/dark mode detection:

