## Available languages

While English (en-US) is the default language, additional languages need to be registered manually before they are ready to use.
Afterwards, the application language is detected and the onyx language is synched automatically so there is no need to manually change it. All listed languages are continuously kept up to date.

::info-card{headline="AI transparency information" color="primary" icon="stars-AI"}
The English and German translations are created manually by native (or near-native) speakers.
Based on them, the other languages are generated using Artificial Intelligence (AI) but might have been (partially) reviewed or adjusted by native or neat-native speakers afterwards.
::

## Installation

::info-card{headline="Editors note" color="neutral"}
Use role switch here to differ between Nuxt and Vue
::

### Vue

Integrate your existing [vue-i18n](https://vue-i18n.intlify.dev/) setup with our translation by passing in your applications locale to the onyx plugin and defining additional languages if needed:

```ts [main.ts]
import { createOnyx } from "sit-onyx";
import onyxDE from "sit-onyx/locales/de-DE.json";
import { createI18n } from "vue-i18n";

const i18n = createI18n({
  // your existing vue-i18n setup...
});

const onyx = createOnyx({
  i18n: {
    locale: i18n.global.locale,
    // the messages locale name here (e.g. "de-DE") MUST be exactly the same as you defined them in your vue-i18n setup (createI18n())
    messages: { "de-DE": onyxDE },
  },
});

createApp(App).use(i18n).use(onyx);
```

Whenever you change your applications locale using vue-i18n, the onyx language will be updated automatically.

### Nuxt

When using Nuxt with our [@sit-onyx/nuxt](/resources/npm-packages/nuxt) module, it will detect which locales you are using in your application and register the corresponding onyx languages automatically.

Only if you are using non 4-digit [BCP 47 language codes](https://developer.mozilla.org/de/docs/Glossary/BCP_47_language_tag), you need to make sure that all your locales define the `language` property correctly:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["@sit-onyx/nuxt", "@nuxtjs/i18n"],
  i18n: {
    locales: [
      { code: "de-DE" },
      // using only "es" as code here, so we need to define the full language manually
      { code: "es", language: "es-ES" },
    ],
  },
});
```
